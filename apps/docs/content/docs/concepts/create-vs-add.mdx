---
title: Create vs Add
description: When to use stackkit create vs stackkit add
---

StackKit has two main commands: `create` for new projects and `add` for enhancing existing ones.

## create

**When to use**: Starting a new project from scratch.

Generates a complete project with your chosen framework, database, and auth. Includes all boilerplate, dependencies, and configurations.

```bash
npx stackkit create my-app --framework nextjs --database prisma-postgresql --auth better-auth
```

**What it does**:
- Creates project directory
- Copies framework template
- Merges selected modules
- Installs dependencies
- Initializes git

## add

**When to use**: Adding features to an existing project.

Detects your current framework and adds modules like auth or database without disrupting your code.

```bash
npx stackkit add
```

**What it does**:
- Detects project type from `package.json`
- Checks compatibility
- Adds files and dependencies
- Updates configurations

## Choosing between them

| Scenario | Command | Example |
|----------|---------|---------|
| New Next.js app with auth | create | `npx stackkit create my-app --framework nextjs --auth better-auth` |
| Add database to existing Express app | add | `npx stackkit add` |
| Prototype React + auth | create | `npx stackkit create my-app --framework react --auth better-auth` |

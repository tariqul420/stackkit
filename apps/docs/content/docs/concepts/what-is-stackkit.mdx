StackKit is a code generator, not a framework. It creates standard projects using popular tools, then gets out of your way.

<Callout type="info">
  Once generated, you own 100% of the code. StackKit is not a runtime dependency.
</Callout>

## Core Concepts

### Templates
Base project structures:
- **Next.js** - Full-stack with App Router
- **Express** - REST API with TypeScript
- **React** - SPA with Vite

### Modules
Optional features:
- **Databases** - Prisma, Mongoose
- **Auth** - Better Auth, Auth.js

### Commands
- `create` - New project
- `add` - Add features
- `doctor` - Verify health
- `list` - Show options

## When to Use

**Good for:**
- New projects
- Team standardization
- Learning full-stack
- Prototypes

**Not for:**
- Custom build configs
- Non-JS/TS projects

## How It Works

### Create Project
```
stackkit create my-app
  ↓
Select options
  ↓
Generate files
  ↓
Install deps
  ↓
Ready!
```

### Add Features
```
stackkit add
  ↓
Detect framework
  ↓
Show modules
  ↓
Install
  ↓
Ready!
```

## Design Principles

### No Lock-in
- You own all code
- No runtime dependency
- Standard tools work
- Deploy anywhere

### Follow Standards
- Framework conventions
- TypeScript strict mode
- ESLint best practices
- Standard env vars

### Smart Defaults
- Everything pre-configured
- Minimal dependencies
- Fast builds
- Type-safe

## What's Included

Every project has:
- TypeScript configuration
- ESLint setup
- Environment variables (.env + .env.example)
- Package scripts
- Git-ready
- Comprehensive README

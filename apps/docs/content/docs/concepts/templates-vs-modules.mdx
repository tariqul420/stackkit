---
title: Templates vs Modules
description: Understanding StackKit's building blocks
---

StackKit uses **templates** for base projects and **modules** for features.

## Templates

Pre-built project structures for frameworks. Include:
- Folder structure
- Config files (TypeScript, ESLint, etc.)
- Basic routing and components
- Development scripts

**Available templates**:
- Next.js (full-stack)
- Express (API)
- React + Vite (frontend)

## Modules

Feature packages that add capabilities. Include:
- Source code files
- Dependencies
- Environment variables
- Database schemas

**Available modules**:
- **Auth**: Better Auth, Auth.js
- **Database**: Prisma, Mongoose

## How they combine

`create-stackkit-app` applies a template + selected modules.

`stackkit-cli` adds modules to existing projects (which may already have a template).

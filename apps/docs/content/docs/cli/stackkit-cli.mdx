## When to use

Adding authentication or database support to an existing project.

## Usage

```bash
npx stackkit-cli <command> [options]
```

<Callout type="info">
Run from your project root (where `package.json` is).
</Callout>

## Commands

### add

Add a module to your project.

```bash
npx stackkit-cli add <module> [options]
```

**Available modules**:
- `better-auth` (auth)
- `authjs` (auth, Next.js + Prisma only)
- `prisma` (database)
- `mongoose` (database)

**Options**:
- `--provider <provider>`: Specify provider (e.g., `postgresql` for Prisma)
- `--force`: Overwrite conflicts
- `--dry-run`: Preview changes
- `--no-install`: Skip dependencies

### list

List available modules for your framework.

```bash
npx stackkit-cli list
```

### doctor

Check project health and compatibility.

```bash
npx stackkit-cli doctor [options]
```

**Options**:
- `--json`: JSON output
- `--verbose`: Detailed info
- `--strict`: Warnings as errors

## Examples

### Add Better Auth

```bash
npx stackkit-cli add better-auth
```

### Add Prisma with PostgreSQL

```bash
npx stackkit-cli add prisma --provider postgresql
```

### Check project health

```bash
npx stackkit-cli doctor
```

## What gets changed

- Adds source files and configs
- Installs dependencies
- Updates environment variables
- Merges schemas and routes

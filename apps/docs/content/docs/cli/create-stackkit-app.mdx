## When to use

Creating a new project from scratch with your preferred stack.

## Commands

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    pnpm create stackkit-app [project-name] [options]
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npx create-stackkit-app [project-name] [options]
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn create stackkit-app [project-name] [options]
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun create stackkit-app [project-name] [options]
    ```
  </Tab>
</Tabs>

## Options

| Flag | Description | Values |
|------|-------------|--------|
| `-f, --framework` | Framework | `nextjs`, `express`, `react-vite` |
| `-d, --database` | Database | `prisma-postgresql`, `prisma-mongodb`, `prisma-mysql`, `prisma-sqlite`, `mongoose-mongodb`, `none` |
| `-a, --auth` | Auth provider | `better-auth`, `authjs`, `none` |
| `-l, --language` | Language | `typescript`, `javascript` |
| `-p, --package-manager` | Package manager | `pnpm`, `npm`, `yarn`, `bun` |
| `--skip-install` | Skip dependency installation | - |
| `--no-git` | Skip git initialization | - |
| `-y, --yes` | Use defaults | - |

## Examples

### Next.js with Prisma and Better Auth

```bash
npx create-stackkit-app my-app --framework nextjs --database prisma-postgresql --auth better-auth
```

### Express API with Mongoose

```bash
npx create-stackkit-app api --framework express --database mongoose-mongodb --auth better-auth
```

### React frontend with auth

```bash
npx create-stackkit-app frontend --framework react-vite --auth better-auth
```

## What gets generated

- Complete project structure
- Framework boilerplate
- Selected modules (auth/database files)
- Dependencies installed
- Git repository initialized
- Environment files

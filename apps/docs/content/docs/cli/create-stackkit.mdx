## Options

All options are optional. Omit them to use the interactive mode.

### `--framework, -f`

Specify the framework:

```bash
npx create-stackkit-app my-app --framework nextjs
```

**Available frameworks:**
- `nextjs` - Next.js 16 with App Router (Full-stack)
- `express` - Express.js REST API (Backend only)
- `react-vite` - React with Vite (Frontend only)

### `--database, -d`

Specify the database adapter:

```bash
npx create-stackkit-app my-app --database prisma-postgresql
```

**Available databases:**
- `prisma-postgresql` - Prisma ORM with PostgreSQL
- `prisma-mongodb` - Prisma ORM with MongoDB
- `mongoose-mongodb` - Mongoose ODM with MongoDB
- `skip` - No database

### `--auth, -a`

Specify the authentication provider:

```bash
npx create-stackkit-app my-app --auth better-auth
```

**Available providers:**
- `better-auth` - Better Auth (TypeScript-first, open source)
- `clerk` - Clerk (hosted authentication service)
- `skip` - No authentication

### `--install`

Install dependencies after project creation (default: `true`).

### `--skip-install`

Skip dependency installation.

### `--git`

Initialize a git repository (default: `true`).

### `--yes, -y`

Use default options without prompts.

### `--help, -h`

Display help information.

## What Gets Generated

The CLI creates a fully configured project:

1. **Framework Setup** - Copies base template with TypeScript, ESLint, and Prettier
2. **Module Integration** - Adds authentication and database code
3. **Environment Configuration** - Creates `.env` files with required variables
4. **Dependency Installation** - Installs all npm packages
5. **Git Initialization** - Initializes repository with `.gitignore`

## Environment Variables

After creation, configure these in your `.env.local` (Next.js) or `.env` (Express):

### Database

```bash
# PostgreSQL
DATABASE_URL="postgresql://user:password@localhost:5432/mydb"

# MongoDB
DATABASE_URL="mongodb://localhost:27017/mydb"
```

### Authentication (Better Auth)

```bash
BETTER_AUTH_SECRET="your-32-char-secret"
BETTER_AUTH_URL="http://localhost:3000"
```

### Authentication (Clerk)

```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."
CLERK_SECRET_KEY="sk_test_..."
```

<Callout type="warn">
  Generate secure secrets using:
  ```bash
  openssl rand -base64 32
  ```
</Callout>

## Common Issues

**Directory already exists**
```
Error: Directory 'my-app' already exists
```
Choose a different name or remove the existing directory.

**Permission denied**
```
Error: EACCES: permission denied
```
Check directory permissions. Never use `sudo` with npm/npx.

**Node version too old**
```
Error: Node version must be >= 20.0.0
```
Update Node.js to version 20 or higher:
```bash
# Using nvm
nvm install 20
nvm use 20
```

## Next Steps

After project creation:

1. **Configure environment** - Update `.env.local` or `.env` with your values
2. **Run migrations** (Prisma only) - `npx prisma migrate dev`
3. **Start dev server** - `npm run dev`
4. **Learn modules** - Read [Authentication](/docs/modules/auth) and [Database](/docs/modules/database) docs

To add modules to existing projects, use [stackkit-cli](/docs/cli/stackkit-cli).

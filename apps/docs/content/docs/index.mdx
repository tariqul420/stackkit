## Why StackKit?

Building a new project shouldn't take hours of setup. StackKit provides:

- **Zero Configuration** - Everything works out of the box
- **Type Safety** - Full TypeScript support across the stack
- **Best Practices** - Industry-standard patterns and folder structure
- **Modern Stack** - Latest versions of Next.js, Prisma, Better Auth, and more
- **Modular** - Add or remove features anytime without breaking code
- **Production Ready** - Deploy-ready configurations included

<Callout type="info">
  **Perfect for:** Hackathons, MVPs, SaaS products, side projects, and production applications
</Callout>

## Example: Next.js + Prisma + Better Auth

Here's what you get with a full-stack Next.js setup:

```typescript title="lib/auth.ts"
// Authentication client pre-configured
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";

export const auth = betterAuth({
  database: prismaAdapter(prisma, {
    provider: "postgresql"
  }),
  emailAndPassword: { enabled: true }
});
```

```typescript title="app/api/auth/[...all]/route.ts"
// API routes ready to use
export { GET, POST } from "@/lib/auth";
```

```prisma title="prisma/schema.prisma"
// Database schema with auth tables
model User {
  id        String   @id @default(cuid())
  email     String   @unique
  name      String?
  sessions  Session[]
}
```

**Start building immediately** - No boilerplate setup required.

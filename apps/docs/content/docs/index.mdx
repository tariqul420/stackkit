StackKit is a modern CLI tool that scaffolds full-stack applications with your choice of framework, database, and authentication—all configured and ready to use in under 2 minutes.

<Callout type="info">
  StackKit generates clean, production-ready code. Once created, you own everything—no vendor lock-in, no runtime dependencies.
</Callout>

## Quick Start

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    pnpm dlx stackkit@latest create my-app
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npx stackkit@latest create my-app
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn dlx stackkit@latest create my-app
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun x stackkit@latest create my-app
    ```
  </Tab>
</Tabs>

## Why StackKit?

### Skip the Boilerplate
No more spending hours setting up TypeScript configs, ESLint rules, or integrating auth libraries. Get a fully configured project instantly.

### Modular by Design
Start minimal or go full-stack. Add databases, authentication, and other features incrementally—only what you need, when you need it.

### Production-Grade Setup
Every project includes TypeScript strict mode, ESLint configuration, environment variable management, and best practices built-in.

### Smart Compatibility
Automatic checks ensure modules work together. No configuration conflicts, no manual debugging of integration issues.

## What You Get

### Templates
- **Next.js** - Full-stack React with App Router
- **Express** - Node.js REST API with TypeScript
- **React** - Frontend SPA with Vite

### Database Integration
- **Prisma** - Modern ORM for PostgreSQL, MySQL, MongoDB, SQLite
- **Mongoose** - Elegant MongoDB ODM
- Pre-configured connection, schema files, and type safety

### Authentication
- **Better Auth** - TypeScript-first auth for all frameworks
- **Auth.js** - Battle-tested auth for Next.js
- Complete setup with routes, middleware, and environment config

### Developer Experience
- Automatic package manager detection
- Git initialization with proper .gitignore
- Environment variable templates
- Health diagnostics with `doctor` command

## Common Workflows

```bash
# Create new project (interactive)
npx stackkit@latest create my-app

# Add features to existing project
npx stackkit@latest add

# Validate project setup
npx stackkit@latest doctor
```

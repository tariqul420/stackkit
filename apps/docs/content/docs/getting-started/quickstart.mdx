Get your first StackKit project running in three simple steps.

<Callout type="info">
  **Requirements**: Node.js 18 or higher
</Callout>

## Step 1: Generate Project

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    pnpm dlx stackkit@latest create my-app
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npx stackkit@latest create my-app
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn dlx stackkit@latest create my-app
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun x stackkit@latest create my-app
    ```
  </Tab>
</Tabs>

## Step 2: Choose Your Stack

Answer a few quick prompts:

1. **Framework** - Next.js, Express, or React
2. **Database** - Prisma, Mongoose, or skip for now
3. **Authentication** - Better Auth, Auth.js, or skip for now
4. **Language** - TypeScript (recommended) or JavaScript
5. **Package Manager** - Auto-detected or choose manually
6. **Git** - Initialize repository (recommended)

StackKit generates everything in seconds.

## Step 3: Start Development

```bash
cd my-app
npm run dev
```

Your application runs at:
- Next.js: http://localhost:3000
- Express: http://localhost:3000
- React: http://localhost:5173

## Project Structure

Generated structure varies by framework. Check your project's README for details.

Common files in all projects:
- `.env.example` - Environment variable template
- `package.json` - Dependencies and scripts
- `tsconfig.json` - TypeScript configuration
- `README.md` - Project-specific documentation

## Configure Environment

Copy `.env.example` to `.env` and fill in your values:

```bash
cp .env.example .env
```

Required variables depend on your selected modules. Check `.env.example` for details.

<Callout type="warn">
  Generate secrets: `openssl rand -base64 32`
</Callout>

## Available Commands

```bash
npm run dev            # Start development
npm run build          # Build for production
npm start              # Run production build
npm run lint           # Lint code
```

Check your project's README for module-specific commands.

## Troubleshooting

### Common Issues

**Port already in use:**
```bash
lsof -ti:3000 | xargs kill -9
```

**Dependencies not working:**
```bash
npm install
```

**Need help?**
```bash
npx stackkit doctor
```

See [Troubleshooting Guide](/docs/reference/troubleshooting) for more solutions.

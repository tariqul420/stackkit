# Installation

This guide walks you through creating your first StackKit project with authentication and database support.

## Prerequisites

- Node.js 20.0 or higher
- Package manager (npm, pnpm, yarn, or bun)
- Git (optional, for version control)

## Create a New Project

Run the interactive CLI to create your project:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    pnpm create stackkit-app my-app
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npx create-stackkit-app my-app
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn create stackkit-app my-app
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun create stackkit-app my-app
    ```
  </Tab>
</Tabs>

## Configure Your Project

The CLI will guide you through project setup:

**Framework Selection:**
- Next.js 16 (App Router) - Full-stack React framework
- Express - Node.js REST API
- React (Vite) - Frontend-only SPA

**Database Selection:**
- Prisma (PostgreSQL) - Recommended for production
- Prisma (MongoDB) - NoSQL with Prisma ORM
- Mongoose (MongoDB) - Traditional MongoDB ODM
- Skip for now

**Authentication Selection:**
- Better Auth - TypeScript-first, open source
- Clerk - Hosted authentication service
- Skip for now

The CLI will then install dependencies and initialize your project.

## Start Development

Navigate to your project and start the development server:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    cd my-app
    pnpm dev
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    cd my-app
    npm run dev
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    cd my-app
    yarn dev
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    cd my-app
    bun dev
    ```
  </Tab>
</Tabs>

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Project Structure

StackKit creates a complete project structure tailored to your framework:

### Next.js Project Structure

```
my-app/
├── app/                    # Next.js App Router
│   ├── (auth)/            # Authentication routes (if selected)
│   ├── api/               # API routes
│   ├── layout.tsx         # Root layout
│   └── page.tsx           # Home page
├── lib/                   # Utility functions
│   ├── auth.ts            # Auth configuration (if selected)
│   └── db.ts              # Database client (if selected)
├── prisma/                # Prisma schema (if selected)
│   └── schema.prisma
├── .env.local             # Environment variables
├── .gitignore
├── next.config.ts
├── package.json
├── tsconfig.json
└── tailwind.config.ts
```

### Express Project Structure

```
my-app/
├── src/
│   ├── routes/            # Express routes
│   ├── middlewares/       # Custom middleware
│   ├── config/            # Configuration files
│   │   ├── auth.ts        # Auth setup (if selected)
│   │   └── db.ts          # Database setup (if selected)
│   ├── app.ts             # Express app setup
│   └── server.ts          # Server entry point
├── .env
├── .gitignore
├── package.json
└── tsconfig.json
```

## Add Modules to Existing Projects

Already have a project? Use `stackkit-cli` to add features:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    # Add authentication
    pnpm dlx stackkit-cli add auth

    # Add database
    pnpm dlx stackkit-cli add database
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    # Add authentication
    npx stackkit-cli add auth

    # Add database
    npx stackkit-cli add database
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    # Add authentication
    yarn dlx stackkit-cli add auth

    # Add database
    ```

</Callout>

## Environment Configuration

Configure environment variables in `.env.local` (Next.js) or `.env` (Express):

**Database:**
```bash
# PostgreSQL
DATABASE_URL="postgresql://user:password@localhost:5432/mydb"

# MongoDB
DATABASE_URL="mongodb://localhost:27017/mydb"
```

**Better Auth:**
```bash
BETTER_AUTH_SECRET="your-secret-key-here"  # Generate: openssl rand -base64 32
BETTER_AUTH_URL="http://localhost:3000"
```

**Clerk:**
```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY="pk_test_..."
CLERK_SECRET_KEY="sk_test_..."
```

<Callout type="warn">
  Never commit `.env` files to version control. They're automatically added to `.gitignore`.
</Callout>

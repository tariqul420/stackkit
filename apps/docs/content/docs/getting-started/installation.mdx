## Create a New Project

Run the interactive CLI to create your project:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    pnpm create stackkit-app my-app
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    npx create-stackkit-app my-app
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn create stackkit-app my-app
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun create stackkit-app my-app
    ```
  </Tab>
</Tabs>

## Configure Your Project

The CLI will guide you through project setup:

**Framework Selection:**
- Next.js 16 (App Router) - Full-stack React framework
- Express - Node.js REST API
- React (Vite) - Frontend-only SPA

**Database Selection:**
- Prisma (PostgreSQL) - Recommended for production
- Prisma (MongoDB) - NoSQL with Prisma ORM
- Mongoose (MongoDB) - Traditional MongoDB ODM
- Skip for now

**Authentication Selection:**
- Better Auth - TypeScript-first, open source
- Skip for now

The CLI will then install dependencies and initialize your project.

## Start Development

Navigate to your project and start the development server:

<Tabs items={['pnpm', 'npm', 'yarn', 'bun']}>
  <Tab value="pnpm">
    ```bash
    cd my-app
    pnpm dev
    ```
  </Tab>
  <Tab value="npm">
    ```bash
    cd my-app
    npm run dev
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    cd my-app
    yarn dev
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    cd my-app
    bun dev
    ```
  </Tab>
</Tabs>

Open [http://localhost:3000](http://localhost:3000) in your browser.

## Project Structure

**Next.js:**
```
my-app/
├── app/          # Next.js App Router
├── lib/          # Auth & database clients
├── prisma/       # Database schema
└── .env.local    # Environment variables
```

**Express:**
```
my-app/
├── src/
│   ├── routes/      # API routes
│   ├── config/      # Auth & database setup
│   └── server.ts
└── .env
```

## Add Modules to Existing Projects

Already have a project? Use `stackkit-cli` to add features:

## Environment Configuration

Configure environment variables in `.env.local` (Next.js) or `.env` (Express):

**Database:**
```bash
# PostgreSQL
DATABASE_URL="postgresql://user:password@localhost:5432/mydb"

# MongoDB
DATABASE_URL="mongodb://localhost:27017/mydb"
```

**Better Auth:**
```bash
BETTER_AUTH_SECRET="your-secret-key-here"  # Generate: openssl rand -base64 32
BETTER_AUTH_URL="http://localhost:3000"
```

<Callout type="warn">
  Never commit `.env` files to version control. They're automatically added to `.gitignore`.
</Callout>

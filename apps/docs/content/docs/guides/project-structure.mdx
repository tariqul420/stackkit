# Project Structure

Understanding the structure of projects generated by StackKit.

## Next.js Project

```
my-app/
├── app/                    # App Router directory
│   ├── (auth)/            # Auth route group (optional)
│   │   ├── sign-in/
│   │   └── sign-up/
│   ├── api/               # API routes
│   │   └── auth/          # Auth endpoints (if Better Auth)
│   ├── globals.css        # Global styles
│   ├── layout.tsx         # Root layout
│   └── page.tsx           # Home page
├── lib/                   # Utilities and config
│   ├── auth.ts            # Auth configuration
│   └── db.ts              # Database client
├── prisma/                # Prisma files (if selected)
│   ├── schema.prisma      # Database schema
│   └── migrations/        # Migration files
├── public/                # Static assets
├── .env.local             # Environment variables
├── .gitignore
├── next.config.ts         # Next.js configuration
├── package.json
├── tailwind.config.ts     # Tailwind CSS config
└── tsconfig.json          # TypeScript config
```

### Key Directories

**`app/`** - Next.js App Router pages and layouts
**`lib/`** - Shared utilities, database clients, and configurations
**`prisma/`** - Database schema and migrations (if using Prisma)
**`public/`** - Static files served from root

## Express Project

```
my-app/
├── src/
│   ├── config/            # Configuration files
│   │   ├── auth.ts        # Auth setup
│   │   └── db.ts          # Database connection
│   ├── middlewares/       # Express middleware
│   │   ├── auth.ts        # Auth middleware
│   │   └── error.ts       # Error handler
│   ├── routes/            # API routes
│   │   ├── auth.ts        # Auth endpoints
│   │   └── index.ts       # Route aggregator
│   ├── models/            # Database models (if Mongoose)
│   ├── app.ts             # Express app setup
│   └── server.ts          # Server entry point
├── prisma/                # Prisma files (if selected)
│   └── schema.prisma
├── .env                   # Environment variables
├── .gitignore
├── package.json
└── tsconfig.json
```

### Key Directories

**`src/config/`** - App configuration and setup
**`src/routes/`** - Express route handlers
**`src/middlewares/`** - Custom middleware functions
**`src/models/`** - Database models (Mongoose)

my-app/
├── src/
│   ├── components/        # React components

├── .env.local            # Environment variables

### Key Directories

**`src/components/`** - Reusable React components
**`src/lib/`** - Utilities and client-side configurations
**`public/`** - Static assets

## Configuration Files

### TypeScript Configuration

All projects include strict TypeScript configuration:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  }
}
```

### Tailwind CSS

Projects use Tailwind CSS for styling:

```typescript title="tailwind.config.ts"
import type { Config } from 'tailwindcss';

const config: Config = {
  content: ['./app/**/*.{js,ts,jsx,tsx}', './components/**/*.{js,ts,jsx,tsx}'],
  theme: {
    extend: {},
  },
  plugins: [],
};

export default config;
```

### Package Scripts

Common scripts available in all projects:

```json title="package.json"
{
  "scripts": {
    "dev": "next dev", // Start dev server
    "build": "next build", // Build for production
    "start": "next start", // Start production server
    "lint": "next lint" // Run ESLint
  }
}
```

## File Naming Conventions

- **React Components**: PascalCase (`UserProfile.tsx`)
- **Utilities**: camelCase (`formatDate.ts`)
- **API Routes**: kebab-case (`/api/user-profile`)
- **Database Models**: PascalCase (`User`, `Post`)
- **CSS Modules**: Component name + `.module.css`

## Import Aliases

Projects use path aliases for cleaner imports:

```typescript
// Instead of
import { db } from '../../../lib/db';

// Use
import { db } from '@/lib/db';
```

Configured in `tsconfig.json`:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./*"]
    }
  }
}
```

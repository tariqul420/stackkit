## Getting Started

### 1. Fork and Clone

```bash
# Fork the repository on GitHub, then:
git clone https://github.com/YOUR_USERNAME/stackkit.git
cd stackkit
```

### 2. Install Dependencies

StackKit uses **pnpm** as the package manager:

```bash
# Install pnpm if you haven't
npm install -g pnpm

# Install dependencies
pnpm install
```

### 3. Build the Project

```bash
# Build all packages
pnpm build
```

## Creating a New Module

Want to add a new authentication provider or database adapter? Follow these steps:

### 1. Read the Module Guide

Start by reading the comprehensive [MODULE_GUIDE.md](https://github.com/tariqul420/stackkit/blob/main/docs/MODULE_GUIDE.md) in the repository.

### 2. Create Module Directory

```bash
# For auth module
mkdir -p modules/auth/your-provider-framework

# For database module
mkdir -p modules/database/your-adapter-framework
```

### 3. Create `module.json`

This file defines your module's metadata and configuration:

```json title="modules/auth/your-provider-nextjs/module.json"
{
  "name": "your-provider-nextjs",
  "displayName": "Your Provider (Next.js)",
  "description": "Authentication with Your Provider for Next.js",
  "category": "auth",
  "provider": "your-provider",
  "supportedFrameworks": ["nextjs"],
  "dependencies": {
    "your-auth-package": "^1.0.0"
  },
  "envVars": [
    {
      "key": "YOUR_PROVIDER_KEY",
      "value": "",
      "description": "API key from Your Provider",
      "required": true
    }
  ],
  "patches": [
    {
      "type": "create-file",
      "description": "Create auth configuration",
      "source": "lib/auth.ts",
      "destination": "{{lib}}/auth.ts"
    }
  ]
}
```

### 4. Add Template Files

Create the files that will be copied to user projects:

```
modules/auth/your-provider-nextjs/
â”œâ”€â”€ module.json
â””â”€â”€ files/
    â”œâ”€â”€ lib/

        â””â”€â”€ api/

                â””â”€â”€ route.ts
```

### 5. Use Placeholders

Use framework-agnostic placeholders in file paths:

| Placeholder  | Next.js | Express      | React |
| ------------ | ------- | ------------ | ----- |
| `{{lib}}`    | `lib`   | `src`        | `src` |
| `{{router}}` | `app`   | `src/routes` | `src` |
| `{{config}}` | `.`     | `.`          | `.`   |

Example in `module.json`:

```json
{
  "patches": [
    {
      "source": "lib/auth.ts",
      "destination": "{{lib}}/auth.ts" // Adapts to framework
    }
  ]
}
```

### 6. Test Your Module

Test with both CLIs:

```bash
# Test with create-stackkit-app
cd apps/create-stackkit
pnpm build
node bin/create-stackkit.js test-project

# Test with stackkit-cli
cd apps/stackkit-cli
pnpm build
cd /path/to/test-project
node /path/to/stackkit-cli/bin/stackkit.js add auth
```

### 7. Submit a Pull Request

1. Create a feature branch: `git checkout -b feat/add-your-module`
2. Commit your changes: `git commit -m "feat: add Your Provider auth module"`
3. Push to your fork: `git push origin feat/add-your-module`
4. Open a Pull Request on GitHub

<Callout title="Pull Request Checklist">
  - [ ] Module works with `create-stackkit-app` - [ ] Module works with `stackkit-cli` - [ ]
  `module.json` is valid - [ ] Template files are tested - [ ] README includes usage instructions -
  [ ] No sensitive data in code
</Callout>

## Code Style

StackKit uses TypeScript with strict type checking:

```bash
# Check types
pnpm typecheck

# Format code
pnpm format

# Lint code
pnpm lint
```

### Conventions

- **Commits**: Use [Conventional Commits](https://www.conventionalcommits.org/)
  - `feat:` - New features
  - `fix:` - Bug fixes
  - `docs:` - Documentation changes
  - `chore:` - Maintenance tasks

- **Naming**: Use kebab-case for files and directories
  - âœ… `better-auth-nextjs`
  - âŒ `BetterAuthNextJs`

- **TypeScript**: Use strict mode, no `any` types

## Testing

### Manual Testing

```bash
# Create test project
cd apps/create-stackkit
pnpm build
node bin/create-stackkit.js ../test-app

# Test CLI tool
cd apps/stackkit-cli
pnpm build
cd ../test-app
node ../stackkit-cli/bin/stackkit.js add auth
```

### Test Checklist

- [ ] Dependencies install correctly
- [ ] Files are created in correct locations
- [ ] Environment variables are added
- [ ] Generated code has no TypeScript errors
- [ ] Project runs with `pnpm dev`
- [ ] Features work as expected

## Documentation

When adding a module, update the documentation:

1. **Module README**: Add usage examples in `modules/your-module/README.md`
2. **Docs Site**: Add page in `apps/docs/content/docs/modules/`
3. **Main README**: Update feature list if needed

## Reporting Issues

Found a bug? Here's how to report it effectively:

### Bug Reports

Use the [bug report template](https://github.com/tariqul420/stackkit/issues/new?template=bug_report.md):

**Required Information:**

- StackKit version
- Node.js version
- Operating system
- Framework (Next.js, Express, React)
- Steps to reproduce
- Expected vs actual behavior

### Feature Requests

Use the [feature request template](https://github.com/tariqul420/stackkit/issues/new?template=feature_request.md):

**Include:**

- Problem you're trying to solve
- Proposed solution
- Alternatives considered
- Examples from other tools
